Start a New Project

1. Follow the documentation to install Node.js. 2. Follow the documentation to install the Angular CLI. Update your global version to the latest version of the cli before creating the new app. 3. Follow the command line instruction to create a new application. It will ask a series of questions about your applications.
- ng new app-name --routing
- Add Angular Routing? Yes [no longer asks, make sure to use –routing for ng commands]
- Stylesheet format? SCSS 4. Cd into the newly created project folder. 5. In the .gitignore file, add package-lock.json. If package-lock.json is already tracked by git, use git rm --cached package-lock.json after updating .gitignore. 6. Add eslint to your project: ng add @angular-eslint/schematics 7. Add Angular Material to your app following the instructions at https://material.angular.io. ng add @angular/material
- Default theme? Indigo/Pink [it doesn’t matter] OR Custom
- Typography? Yes
- Setup Browser Animations? Yes 8. Add Prettier to your application 9. Create a module under shared for all of the material pieces. Comment them out initially, them add them back in as needed. To get the list, go to the instructions for one of the components, then to the examples and copy and paste from there. For example: ng generate module shared\modules\material\
Then go to a component on https://material.angular.io/ and view its examples. Copy the imports into the module you just created. Comment them all out and uncomment as needed. Cleanup code if module is not the right name, etc. 10. Create your modules, components, services, etc. with the cli: ng generate TypeOfItem name-of-item

11. Add a .htaccess file to the project to handle url rewrites. Check   https://angular.io/guide/deployment   for updates.

12. Make files in src/assets/images and src/assets/css available without needing to use weird relative paths in angular.json.
